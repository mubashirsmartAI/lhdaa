Order Panel – Easy Guide

This page explains in very simple words what this project does, how it is organised, and how you can open all the panels. No technical background is required.

What this project does

1. Store Website

Visitors can browse products, categories and promotions. Everything you see on the public site comes from the database and is shown by Laravel pages.

2. Client/Admin Panel

Business owners log in here to manage orders, products, vendors, etc. Users are stored in a table called “users”.

3. God Panel

This is a super-admin area for the platform provider. It manages client accounts, subscriptions and billing. Users are stored in a table called “admins”.

4. Shared Services

Helpers keep track of client settings, images, currencies and caching. Middleware can switch databases per client (disabled while working locally).

How the website answers a request

1. Browser opens a URL → 2. Laravel route decides which controller runs → 3. Middleware loads client settings & sessions → 4. Controller reads data from database → 5. Blade view or JSON is returned → 6. Browser shows HTML + JS (AOS/custom.js)

How to run everything locally

  1. Install PHP 7.4 + Composer (already done in this setup).
  2. Copy .env.example to .env and add your MySQL info (Laragon default: DB new_project, user root, no password).
  3. Run: composer install (already done) and php artisan key:generate.
  4. Run migrations + seeders: php artisan migrate --seed.
  5. Serve the site: php artisan serve --host=127.0.0.1 --port=1500.
  6. Open a browser:
    • Storefront: http://127.0.0.1:1500/
    • Client/Admin login: http://127.0.0.1:1500/client/login
    • God panel login: http://127.0.0.1:1500/godpanel/login

What to do after logging in

Client/Admin Panel

  1. Dashboard shows quick stats.
  2. Use menu to add categories, vendors, products.
  3. Orders tab lets you accept/track orders.
  4. Settings tab controls currencies, taxes, preferences.

God Panel

  1. View list of clients (businesses).
  2. Create new client accounts, assign plans.
  3. Manage subscriptions/payments.
  4. Switch into a client account if needed.

Glossary (non-technical)

Controller

A PHP file that decides what data to load for a page. Example: UserhomeController loads home page data.

Route

A line in the routes folder that says “when someone visits /client/login, run this controller method”.

Seeder

A script that inserts default records into the database (like default admin users).

Blade View

HTML templates stored in resources/views. They display the data from controllers.
